    POST
    https://api.bring.com/booking/api/booking

Request body

Body param Type Description
testIndicator boolean Required It is true for test orders.
schemaVersion integer Required Currently, it is 1
consignments array Required A list of consignment objects
shippingDateTime dateTime Required The date and time when an order is ready to be transported. The label generated against a booking will be valid for 52 weeks when Bring receives the request for an order.
Example: 2016-07-26T06:24:09 or 1469494449000 (milliseconds since 01.01.1970, for json req)
customerSpecifiedDeliveryDateTime dateTime Optional The date and time selected for preferred delivery by the customer. Note that for some services, the time part of this field will be ignored. This field is applicable for ‘PA_DOREN’ ‘OIL_EXPRESS’.
parties object Required List of parties: sender, recipient, returnTo, pickupPoint and importer
name string Required Specify name of sender or receiver
Example: Ola Nordmann
addressLine string Required Specify address. This parameter can be maximum 35 character long.
Example: Testsvingen 12
addressLine2 string Optional Specify address line 2. This parameter can be 35 character long.
postalCode string Required Specify the postal code. This parameter can be maximum 35 character long.
city string Required Specify the city. This parameter can be maximum 35 character long.
countryCode string Required Specify the country code. This parameter must be 2 characters long.
reference string Optional Specify shipper or consignee reference. This parameter can be maximum 35 character long.
additionalAddressInfo string Optional Pickup instruction and delivery instruction. This parameter can be maximum 35 character long.
vatNumber string Optional Specify organisation number. This parameter can be maximum 30 character long.
contact object Optional Specify contact information
name string Optional Name of sender or receiver
phoneNumber string Optional Specify phone number.
Note: Sender Phone number is a mandatory field for all outgoing services. Exceptions to this rule are 3584, 3570 and all the return services in Mybring.
email string Optional Specify e-mail address if you want booking confirmation e-mail to be sent. Can be applied to sender’s e-mail (outgoing services) and sender’s e-mail/recipient’s e-mail (return services). This parameter can be maximum 35 character long.
Pattern: ^((\w*_)?(p|m|)\d*(\.\d|)|(\w*_)?(p|m)\d*(\.\d|)p)$
id string Required for Pickup point Pickup point id. Use Pickup Point API to get list of nearest pickup points.
Example: 736431
countryCode string Required for Pickup point Specify the country code. This parameter must be 2 characters long.
Example: NO
product object Required List of service related information
id string Required Product id. You can list available products with the /customers endpoint.
Example: BPAKKE_DOR-DOR
customerNumber string RequiredSpecify API customer number. This parameter can be maximum 35 character long.
correlationId string Optional Specify something to correlate packages which belong to same order
packages array Optional List of package specific information.
weightInKg double Optional Specify weight in kilograms
Example: 15.2
volumeInDm3 decimal Optional Specify volume in cubic decimeter. This is optional when dimension is specified in the request for cargo.
dimensions object Optional List of package dimension properties
heightInCm integer Optional Specify height in cm. Dimension is optional when volumeInDm3 is specified in the request for cargo.
Example: 100
widthInCm integer Optional Specify width in cm. Dimension is optional when volumeInDm3 is specified in the request for cargo.
Example: 100
lengthInCm integer Optional Specify length in cm. Dimension is optional when volumeInDm3 is specified in the request for cargo.
Example: 100
generateQrCodes boolean Optional Specify whether QR codes is wanted in addition to the regular label for certain services. See the qr-codes section for more on this.
additionalServices array Optional List of value added services
id string Optional The ID of the value added service
